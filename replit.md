# Fokus ‚Äî Daily Focus Widget App

A React Native (Expo) mobile app for setting a daily focus that appears on your iPhone home screen as a native iOS widget.

## Run & Operate

- `pnpm --filter @workspace/widget-app run dev` ‚Äî run the Expo dev server (scan QR in Expo Go)
- `pnpm --filter @workspace/api-server run dev` ‚Äî run the API server (port 5000)
- `pnpm run typecheck` ‚Äî full typecheck across all packages
- `pnpm run build` ‚Äî typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` ‚Äî regenerate API hooks and Zod schemas from the OpenAPI spec

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Mobile: Expo ~54 + Expo Router, React Native 0.81
- State: React Context + AsyncStorage (no backend needed)
- API: Express 5 (shared backend)
- Build: esbuild (CJS bundle)

## iOS Widget Architecture

The native widget requires a production build (Expo Launch):

1. **Config plugin** (`plugins/withFokusWidget.js`) ‚Äî adds App Group entitlement, copies Swift files into `ios/`, adds widget extension target to Xcode project, and registers a native module (`FokusWidgetBridge`) for data writing.
2. **Swift widget** (`target/FokusWidget/FokusWidget.swift`) ‚Äî WidgetKit extension that reads JSON from shared UserDefaults (`group.com.replit.fokus`) and renders Small / Medium / Large views in SwiftUI.
3. **Native bridge** ‚Äî `FokusWidgetBridge.m` (generated by plugin) exposes `updateData(json)` to React Native, writes to shared UserDefaults, and calls `WidgetCenter.shared.reloadAllTimelines()`.
4. **Graceful fallback** ‚Äî In Expo Go, `FocusContext` stores data in AsyncStorage only. The widget data sync activates automatically in the native build.

## App Features

- Set a daily focus with category: Today's Focus, Goal, Quote, or Reminder
- Live widget preview in Small / Medium / Large sizes (indigo gradient)
- Focus history with category filtering
- Tap any history item to make it active again
- Step-by-step guide for adding widget to iPhone home screen
- Settings screen with history management

## Architecture decisions

- Frontend-only by default: no database, all data in AsyncStorage; widget bridge activates transparently in native builds
- Config plugin approach for widget: avoids third-party native libraries while giving full control over the Xcode target setup
- FocusContext manages both current item and capped history (20 items max)
- Widget preview component mirrors actual iOS widget proportions (155pt small, 329√ó155 medium, 329√ó345 large)
- Graceful NativeModules fallback: `tryUpdateNativeWidget` wraps the bridge call in try/catch so Expo Go never crashes

## Product

Users set a short daily focus (task, quote, goal, or reminder) in the app. It appears on their iPhone home screen widget in three sizes, updating automatically every 30 minutes. History lets users reuse past focuses instantly.

## Gotchas

- Widget requires native build via Expo Launch ‚Äî it won't appear from Expo Go alone
- App Group ID is `group.com.replit.fokus` ‚Äî must match in `app.json`, config plugin, and Swift Info.plist
- Never change the bundle identifier (`com.replit.fokus`) after a build is submitted
- Do not create `app.config.ts` ‚Äî Expo Launch requires a static `app.json`
- Widget refreshes on a ~30-min system-controlled schedule; `reloadAllTimelines()` in the bridge triggers an immediate refresh when focus changes
